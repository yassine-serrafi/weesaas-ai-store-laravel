De la photo à la vente,
en quelques minutes.

WeeSaaS est une plateforme e-commerce qui transforme une simple image produit en page de vente complète — l'IA rédige, shoote en studio, traduit et publie. Cette documentation décrit l'architecture, le fonctionnement et chaque module du projet.

Laravel 12 PHP 8.2+ MySQL / MariaDB Gemini · GPT-4o COD multilingue ar/fr/en

🧭 Vue d'ensemble

WeeSaaS génère des landing pages produit orientées conversion à partir d'une seule photo, et gère tout le cycle de vente en paiement à la livraison (COD), avec un panneau d'administration complet.

🔐 Sécurité

🔑

Auth admin

bcrypt · anti-bruteforce (5 essais / 15 min par IP) · sessions liées à l'IP + expiration · régénération anti-fixation.

🛡️

CSRF

Actif partout (seul /track exempté pour sendBeacon).

🧱

En-têtes

CSP, HSTS (HTTPS), Permissions-Policy, X-Frame-Options, X-Content-Type-Options, Referrer-Policy.

🚦

Rate-limiting

Throttle sur les endpoints publics (commande, formulaires, tracking).

🧬

Anti-injection

Eloquent partout ; valeurs castées dans les rares requêtes brutes.

📤

Uploads

Validés (jpg/png/webp uniquement, taille limitée — SVG bloqué).

⚠️
Avant la prod : APP_ENV=production, APP_DEBUG=false, et supprimer public/install.php (l'installateur s'auto-supprime).

⚙️ File d'attente & Jobs

La génération est asynchrone pour ne pas bloquer l'interface.

  • GenerateProductPage — Job qui exécute toute la génération (analyse, textes, images, sauvegarde)
  • Driver de file : database ; un worker démarre automatiquement au clic (QueueRunner)
  • Logs en direct via generation_logs + suivi d'état (ai_jobs)
  • Sur hébergement mutualisé : un cron queue:work prend le relais si le worker auto est bloqué

📦 Installation

🖱️

Assistant web

Ouvrir tondomaine.com/install.php : vérifie le serveur, configure la base, crée l'admin, puis s'auto-supprime. Guide cPanel : INSTALL.md.

💻

Développement

composer install → cp .env.example .env → php artisan key:generate → php artisan migrate → php artisan serve.

# Tout-en-un (dev) : serveur + file + logs + assets
composer run dev
🔐
L'admin se connecte sur /weeadmin avec le compte créé à l'installation. Renseigner ensuite les clés API IA dans Réglages.
